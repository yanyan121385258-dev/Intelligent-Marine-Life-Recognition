# Hertz Studio Django 系统流程设计文档

## 一、系统概述

### 1.1 系统简介
Hertz Studio Django 是一个基于 Django 5 + Vue 3 + TypeScript 构建的统一后端服务平台，面向 AI 工作室与通用后台场景。系统采用模块化设计，提供 REST API 与 WebSocket 双协议支持，集成了认证权限、通知公告、日志管理、知识库、系统监控、AI 对话、代码生成、YOLO 目标检测等多个功能模块。

### 1.2 系统目标
- 提供统一的用户认证与权限管理平台
- 实现 AI 工作室的数字化管理
- 提供 YOLO 目标检测服务
- 集成 AI 对话与知识库功能
- 支持代码生成与系统监控
- 提供完善的日志管理与审计功能

### 1.3 技术栈
- **后端框架**: Django 5 + Django REST Framework + Channels + Daphne
- **前端框架**: Vue 3 + TypeScript + Vite + Ant Design Vue
- **数据库**: SQLite (默认) / MySQL (可选)
- **缓存**: Redis (缓存、会话、Channel Layer)
- **API 文档**: drf-spectacular (Swagger + Redoc)
- **认证方式**: JWT (pyjwt) + 自定义 AuthMiddleware
- **AI/ML**: Ultralytics YOLO、OpenCV、NumPy、Scikit-learn、Ollama

## 三、用户角色与权限设计

### 3.1 角色定义

#### 3.1.1 超级管理员 (Super Admin)
**角色代码**: `super_admin`
**权限范围**: 系统最高权限

**功能权限**:
- 用户管理: 所有用户的管理权限
- 角色管理: 角色的创建、编辑、删除、权限分配
- 菜单管理: 菜单的创建、编辑、删除、权限设置
- 部门管理: 部门的创建、编辑、删除
- 系统配置: 系统参数配置
- 日志管理: 所有日志的查看和管理
- 通知管理: 通知公告的管理
- 知识库管理: 知识文章的管理
- YOLO 模型管理: 模型的上传、启用、删除
- 系统监控: 系统指标的查看
- AI 对话: AI 对话功能的使用

#### 3.1.2 管理员 (Admin)
**角色代码**: `admin`
**权限范围**: 部分管理权限

**功能权限**:
- 用户管理: 普通用户的管理
- 日志管理: 日志查看
- 通知管理: 通知公告的创建、编辑
- 知识库管理: 知识文章的创建、编辑
- YOLO 模型管理: 模型的查看、使用
- 系统监控: 系统指标的查看
- AI 对话: AI 对话功能的使用

#### 3.1.3 普通用户 (User)
**角色代码**: `user`
**权限范围**: 基础功能权限

**功能权限**:
- 个人中心: 个人信息管理、密码修改
- 通知查看: 查看通知公告
- 知识库查看: 查看知识文章
- YOLO 检测: 使用 YOLO 检测功能
- AI 对话: 使用 AI 对话功能
- 系统监控: 查看系统指标

        # 验证 JWT Token
