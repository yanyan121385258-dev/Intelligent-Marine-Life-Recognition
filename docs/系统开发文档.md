# 智慧海洋生物识别系统 - 开发文档

## 1. 系统概述

**智慧海洋生物识别系统**是一个基于 Django + Vue3 构建的综合性海洋生物监测与识别平台，集成了用户认证、AI检测、知识库管理等核心功能模块。

### 1.1 技术栈

| 分类 | 技术 | 版本 |
|------|------|------|
| 后端框架 | Django | 5.2.6 |
| 前端框架 | Vue | 3.x |
| UI组件库 | Ant Design Vue | 3.x |
| 构建工具 | Vite | 6.x |
| 数据库 | SQLite/MySQL | - |
| 缓存 | Redis | 6.x+ |
| 实时通信 | WebSocket (Channels) | - |
| API文档 | DRF Spectacular | - |

## 2. 核心功能模块

### 2.1 权限认证模块 (`studio_django_auth`)

管理员进入权限认证模块后，主界面以列表形式清晰展示所有系统用户信息，包括用户名、邮箱、真实姓名、手机号、所属部门、账户状态及注册时间等核心字段。顶部提供按用户名、邮箱的模糊查询功能，并支持按账户状态（活跃/禁用）和用户角色进行快速筛选。针对每位用户，管理员可点击详情查看完整的账户信息及关联角色，并执行编辑操作修改用户基本信息、分配或变更用户角色；同时支持对异常账户进行禁用或删除操作，确保系统用户管理的规范与安全。

#### 用户管理

| 功能 | API接口 | 权限标识 |
|------|---------|----------|
| 用户列表 | `GET /api/auth/users/` | `system:user:list` |
| 创建用户 | `POST /api/auth/users/` | `system:user:add` |
| 用户详情 | `GET /api/auth/users/{id}/` | `system:user:query` |
| 更新用户 | `PUT /api/auth/users/{id}/` | `system:user:edit` |
| 删除用户 | `DELETE /api/auth/users/{id}/` | `system:user:remove` |
| 分配角色 | `POST /api/auth/users/assign-roles/` | `system:user:role` |

#### 角色管理

管理员进入角色管理模块后，主界面以列表形式清晰展示系统所有角色信息，包括角色名称、角色编码、角色描述、关联菜单数量、排序权重及创建时间等核心字段。顶部提供按角色名称、角色编码的模糊查询功能。针对每个角色，管理员可点击详情查看完整的角色信息及所分配的菜单权限，并执行编辑操作修改角色基本信息、调整角色关联的菜单权限范围；同时支持新增角色或对无需保留的角色进行删除操作，确保系统权限分配的灵活与可控。

| 功能 | API接口 | 权限标识 |
|------|---------|----------|
| 角色列表 | `GET /api/auth/roles/` | `system:role:list` |
| 创建角色 | `POST /api/auth/roles/` | `system:role:add` |
| 角色详情 | `GET /api/auth/roles/{id}/` | `system:role:query` |
| 更新角色 | `PUT /api/auth/roles/{id}/` | `system:role:edit` |
| 删除角色 | `DELETE /api/auth/roles/{id}/` | `system:role:remove` |
| 分配菜单 | `POST /api/auth/roles/assign-menus/` | `system:role:menu` |

#### 菜单管理

管理员进入菜单管理模块后，主界面以树形结构清晰展示系统完整的功能菜单层级，包括菜单名称、菜单编码、菜单类型（目录/菜单/按钮）、路由路径、组件路径、菜单图标、权限标识及排序权重等核心字段。顶部提供按菜单名称的模糊查询功能，并支持按菜单类型进行快速筛选。针对每个菜单项，管理员可点击详情查看完整的菜单配置信息，并执行编辑操作修改菜单名称、调整路由路径或组件配置；同时支持新增同级或子级菜单，或对无需保留的菜单进行删除操作，确保系统导航结构的清晰与有序。

| 功能 | API接口 | 权限标识 |
|------|---------|----------|
| 菜单列表 | `GET /api/auth/menus/` | `system:menu:list` |
| 菜单树 | `GET /api/auth/menus/tree/` | `system:menu:list` |
| 创建菜单 | `POST /api/auth/menus/` | `system:menu:add` |
| 更新菜单 | `PUT /api/auth/menus/{id}/` | `system:menu:edit` |
| 删除菜单 | `DELETE /api/auth/menus/{id}/` | `system:menu:remove` |

#### 部门管理

管理员进入部门管理模块后，主界面以树形结构清晰展示系统的组织架构层级，包括部门名称、部门编码、部门负责人、联系电话、邮箱及排序权重等核心字段。顶部提供按部门名称的模糊查询功能。针对每个部门，管理员可点击详情查看完整的部门信息，并执行编辑操作修改部门基本信息、调整上级部门或变更部门负责人；同时支持新增同级或子级部门，或对无关联用户和子部门的部门进行删除操作，确保组织架构的规范与完整。

| 功能 | API接口 | 权限标识 |
|------|---------|----------|
| 部门列表 | `GET /api/auth/departments/` | `system:dept:list` |
| 部门树 | `GET /api/auth/departments/tree/` | `system:dept:list` |
| 创建部门 | `POST /api/auth/departments/` | `system:dept:add` |
| 更新部门 | `PUT /api/auth/departments/{id}/` | `system:dept:edit` |
| 删除部门 | `DELETE /api/auth/departments/{id}/` | `system:dept:remove` |

### 2.2 验证码模块 (`studio_django_captcha`)

用户进入验证码模块后，系统自动生成包含干扰线条和噪点的验证码图片，展示用户名、邮箱的输入框及验证码填写区域。系统支持通过刷新按钮重新获取新的验证码图片，用户需在规定时间内正确识别并填写验证码后方可完成注册或登录操作，有效防止自动化恶意攻击。

| 功能 | API接口 | 说明 |
|------|---------|------|
| 获取验证码 | `GET /api/captcha/image/` | 返回验证码图片和key |
| 验证验证码 | `POST /api/captcha/verify/` | 验证用户输入 |

### 2.3 YOLO目标检测模块 (`studio_django_yolo`)

用户进入YOLO目标检测模块后，主界面以卡片形式清晰展示已加载的检测模型信息，包括模型名称、模型版本、检测精度及运行状态等核心指标。用户提供图片上传入口，系统自动调用YOLO模型进行目标检测，并以可视化框选形式在原图上标注检测到的海洋生物目标，显示目标类别名称和置信度。针对每条检测记录，系统自动保存至检测历史，管理员可点击详情查看完整的检测结果、标注图片及检测参数配置；同时支持查看所有检测历史记录、筛选不同模型或时间范围的检测数据，确保海洋生物检测工作的高效与可追溯。

#### 检测功能

| 功能 | API接口 | 说明 |
|------|---------|------|
| 单图检测 | `POST /api/yolo/detect/` | 上传图片进行检测 |
| 批量检测 | `POST /api/yolo/batch-detect/` | 批量图片检测 |
| 实时检测 | WebSocket `/websocket/yolo/` | 实时视频流检测 |
| 检测历史 | `GET /api/yolo/history/` | 查询检测记录 |

#### 模型管理

管理员进入模型管理模块后，主界面以列表形式清晰展示所有可用的YOLO检测模型，包括模型名称、模型版本、训练数据集、检测精度（mAP）、模型大小及加载状态等核心字段。顶部提供按模型名称的模糊查询功能，并支持按模型类型进行快速筛选。针对每个模型，管理员可点击详情查看完整的模型训练参数、配置文件及性能评估报告，并执行模型加载或卸载操作将模型部署至检测服务；同时支持查看模型关联的训练记录和检测统计数据，或对无需保留的模型执行删除操作，确保检测服务的模型管理规范与高效。

| 功能 | API接口 | 说明 |
|------|---------|------|
| 模型列表 | `GET /api/yolo/models/` | 获取可用模型 |
| 模型详情 | `GET /api/yolo/models/{id}/` | 模型信息 |
| 加载模型 | `POST /api/yolo/models/{id}/load/` | 加载到内存 |
| 卸载模型 | `POST /api/yolo/models/{id}/unload/` | 从内存卸载 |

| 功能 | API接口 | 说明 |
|------|---------|------|
| 数据集列表 | `GET /api/yolo/train/datasets/` | 训练数据集管理 |
| 创建数据集 | `POST /api/yolo/train/datasets/` | 上传训练数据 |
| 训练任务 | `POST /api/yolo/train/tasks/` | 创建训练任务 |
| 任务状态 | `GET /api/yolo/train/tasks/{id}/` | 查询训练进度 |
| 停止训练 | `POST /api/yolo/train/tasks/{id}/stop/` | 终止训练 |

### 2.5 AI聊天模块 (`studio_django_ai`)

用户进入AI聊天模块后，主界面以对话气泡形式清晰展示与AI助手的历史对话记录，包括用户提问内容、AI回复内容、对话时间及关联知识库来源等核心信息。用户提供文本输入框发送消息，系统基于知识库内容进行智能问答，并支持在回复中展示引用的知识库文档链接。用户可对不满意的历史回复进行反馈标记，或清空对话重新开始新的咨询；同时支持查看所有对话历史记录、筛选不同时间范围的对话数据，确保AI知识问答服务的便捷与高效。

| 功能 | API接口 | 说明 |
|------|---------|------|
| 发送消息 | `POST /api/ai/chat/` | 与AI对话 |
| 消息历史 | `GET /api/ai/chat/history/` | 获取对话记录 |
| 清除历史 | `DELETE /api/ai/chat/history/` | 清空对话 |
| 知识库检索 | `POST /api/ai/kb-search/` | 基于知识库回答 |

## 5. 认证与权限机制

### 5.1 JWT认证流程

```
1. 用户登录 → 获取 JWT Token
2. 请求API → 携带 Token (Authorization: Bearer xxx)
3. AuthMiddleware → 验证Token有效性
4. 权限检查 → 验证用户是否有权限访问
5. 返回响应
```

### 5.2 权限控制

- **超级管理员**: 拥有所有权限 (`*`)
- **角色权限**: 通过角色关联的菜单权限控制
- **权限标识格式**: `{module}:{resource}:{action}`
  - 示例: `system:user:list`, `system:role:add`

### 5.3 免认证URL

以下URL不需要登录即可访问：
- `/api/auth/login/` - 登录
- `/api/auth/register/` - 注册
- `/api/captcha/*` - 验证码
- `/api/docs/*` - API文档
- `/demo/*` - 演示接口
