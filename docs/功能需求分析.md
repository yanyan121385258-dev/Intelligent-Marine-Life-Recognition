# Hertz Studio Django 功能需求分析文档

## 一、项目概述

### 1.1 项目简介
Hertz Studio Django 是一个基于 Django 5 + Vue 3 + TypeScript 构建的统一后端服务平台，面向 AI 工作室与通用后台场景。系统采用模块化设计，提供 REST API 与 WebSocket 双协议支持，集成了认证权限、通知公告、日志管理、知识库、系统监控、AI 对话、代码生成、YOLO 目标检测等多个功能模块。

### 1.2 技术架构
- **后端框架**: Django 5 + Django REST Framework + Channels + Daphne
- **前端框架**: Vue 3 + TypeScript + Vite + Ant Design Vue
- **数据库**: SQLite (默认) / MySQL (可选)
- **缓存**: Redis (缓存、会话、Channel Layer)
- **API 文档**: drf-spectacular (Swagger + Redoc)
- **认证方式**: JWT (pyjwt) + 自定义 AuthMiddleware
- **AI/ML**: Ultralytics YOLO、OpenCV、NumPy、Scikit-learn、Ollama

### 1.3 系统角色
- **超级管理员**: hertz / hertz (拥有所有权限)
- **普通用户**: demo / 123456 (拥有基础功能权限)

## 二、功能模块详细需求

### 2.1 认证与权限模块 (studio_django_auth)

#### 2.1.1 用户认证功能
**功能需求:**
1. **用户登录**
   - 支持用户名/密码登录
   - 集成图形验证码验证
   - 返回 JWT access_token 和 refresh_token
   - 返回用户基本信息、角色、权限列表

2. **用户注册**
   - 支持用户自主注册
   - 必填字段: 用户名、密码、确认密码、邮箱、手机号、真实姓名
   - 可选字段: 邮箱验证码
   - 密码强度验证
   - 邮箱格式验证
   - 手机号格式验证

3. **用户登出**
   - 清除用户会话信息
   - 使 JWT token 失效

4. **Token 刷新**
   - 使用 refresh_token 刷新 access_token
   - 延长用户会话时间

5. **密码管理**
   - 修改密码 (需验证旧密码)
   - 重置密码 (通过邮箱验证码)
   - 发送邮箱验证码

#### 2.1.2 用户管理功能

2. **用户信息管理**

## 三、非功能性需求

### 3.1 性能需求
1. **响应时间**
   - API 接口响应时间 < 500ms
   - 页面加载时间 < 2s
   - 文件上传支持大文件 (支持分片上传)

2. **并发能力**
   - 支持 100+ 并发用户
   - WebSocket 支持 50+ 并发连接

3. **数据处理**
   - 支持大数据量查询 (分页)
   - 支持批量操作

### 3.2 安全需求
1. **认证安全**
   - JWT Token 认证
   - Token 过期时间控制
   - 密码加密存储

2. **权限控制**
   - 基于角色的权限控制 (RBAC)
   - 接口级权限验证
   - 菜单级权限控制

3. **数据安全**
   - SQL 注入防护
   - XSS 攻击防护
   - CSRF 防护
   - CORS 配置

4. **日志审计**
   - 操作日志记录
   - 登录日志记录
   - 异常日志记录

### 3.3 可用性需求
1. **用户界面**
   - 响应式设计
   - 支持多浏览器 (Chrome、Firefox、Edge)
   - 支持移动端访问

2. **用户体验**
   - 友好的错误提示
   - 操作引导
   - 快捷键支持

3. **国际化**
   - 支持中文
   - 支持英文 (可扩展)

### 3.4 可维护性需求
1. **代码规范**
   - 遵循 PEP 8 (Python)
   - 遵循 ESLint (JavaScript/TypeScript)
   - 代码注释完整

2. **文档完善**
   - API 接口文档
   - 使用手册
   - 开发规范

3. **模块化设计**
   - 功能模块独立
   - 易于扩展
   - 易于维护

### 3.5 可扩展性需求
1. **插件机制**
   - 支持自定义模块

2. **配置灵活**

## 七、系统流程

### 7.1 用户登录流程
```
1. 用户输入用户名、密码、验证码
2. 系统验证验证码
3. 系统验证用户名和密码
4. 系统生成 JWT Token
5. 系统返回 Token 和用户信息
6. 前端保存 Token 到本地存储
7. 前端跳转到首页
```

### 7.2 权限验证流程
```
1. 用户访问受保护资源
2. 系统检查请求头中的 Token
3. 系统验证 Token 有效性
4. 系统解析用户信息
5. 系统检查用户权限
6. 如果有权限，允许访问
7. 如果无权限，返回 403 错误
```

### 7.3 YOLO 检测流程
```
1. 用户上传图片/视频
2. 系统接收文件
3. 系统加载 YOLO 模型
4. 系统执行目标检测
5. 系统绘制检测结果
6. 系统保存结果图片/视频
7. 系统返回检测结果
8. 系统保存检测历史
```

### 7.4 AI 对话流程
```
1. 用户发送消息
2. 系统接收消息
3. 系统调用 Ollama API
4. 系统接收 AI 回复
5. 系统通过 WebSocket 推送回复
6. 系统保存对话历史
```
